# Processed data

Small derived tables — enough to reproduce every figure and metric in the paper
without rerunning the weather models or LISFLOOD-FP. All discharge in m³ s⁻¹, depth/
level in m, times UTC.

## Per-gauge time series — `<model>_station_Q.csv`
One file per forcing (`wrf`, `wrf12`, `graphcast`, `fuxi`, `aifs`). Columns:
`Time` plus `<Gauge>_Q` and `<Gauge>_Depth` for each of Walferdange, Steinsel,
Pfaffenthal, Livange, Hesperange. 6-hourly; idx 0 = 2021-07-13 00:00 UTC.
The four complete runs cover idx 0–17 (102 h); GraphCast covers idx 0–16 (it reached
97 h before the 18 h job wall), which still spans the full 96 h analysis window.

## Figure tables — `pgfplots/`
- `<gauge>_merged.csv` — observed + all seven forcings on the common 6-hourly grid
  (idx 0–16, the **96 h analysis window** 13 Jul 00:00 → 17 Jul 00:00 UTC). EFAS
  benchmarks (`EFAS_ECMWF`, `EFAS_DWD`) are discharge-only and `nan` where the
  companion EFAS series ends.
- `walferdange_merged.csv` — Walferdange water level / depth (m); EFAS omitted.
- `averaged_metrics.csv` — skill (NSE, KGE, PDE, PTE) and error (RMSE, MAE, SMAPE,
  Bias) averaged across the four discharge gauges, one row per model.

These are regenerated by `../../analysis/plot_and_stats.py`. Released under CC-BY-4.0.
